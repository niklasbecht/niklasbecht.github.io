Swarm Sheperd XR - WIP

In this VR project, the player controls a flock of Boids using a VR controller through button interactions, guiding them to targets, avoiding obstacles, and interacting with the environment. The flock behavior is based on a compute shader implementation of classic Boid rules (cohesion, alignment, separation) that allows real-time obstacle avoidance and dynamic density adjustments. The goal of the project is to gain experience with compute shaders and to recreate an older CPU-based 2D project in 3D VR. Key gameplay features include flock attraction to the controller, spreading/separation on command, and environment-based puzzle interactions, all within an immersive VR experience.

Implementation and parameterization of VR-exergame-mechanics

In this project, three different prototypes were developed for various VR exergame mechanics. Our objective behind these prototypes was to empower users to manage their heart rate, keeping it within a predefined range, while ensuring its seamless integration into gameplay. During a training session, all relevant actions and heart rate information are monitored and recorded, allowing the session to be observed by an external person and later on evaluated. User tests confirmed the effectiveness of our three VR exergame prototypes in maintaining heart rates within a targeted range for individuals with heart failure. The unique feature of these games is their parameterization, which allows personal customization based on real-time heart rate data, further enhancing the immersive gaming experience and encouraging the user towards a more active lifestyle.

  • Fight Dummy
    The user interacts with the Fight Dummy, while the interaction between the dummy and the user adapts to the user's heart rate.
  • Endless Climb
    The user scales an endless k, while an artificial endurance system adapts to the user's heart rate and forces him to perform more authentic and exhausting climbing movements.
  • Water Lock
    Water Lock is the third prototype, the user is placed in a virtual kayak. This kayak floats within a lock on a dynamic water surface. The user paddles against an adaptive counter-current and interacts with various objects. The prototype automatically adjusts the strength of the counter-current based on the user's heart rate to provide optimal exertion.

VR · Locomotion · Unity

Physics-based Character Controller for OpenXR

This project covers design and development of a physics-based character controller for VR. With increasing popularity and availability of VR hardware, the diversity of available hardware and software also increases. The OpenXR standard provides an abstraction layer that allows programming in a hardware- and software-independent manner. The Unity engine offers an implementation of this standard and provides the component-based XR Interaction Toolkit. This toolkit enables interactions and locomotion in VR and AR applications. Locomotion, also known as movement, in a virtual environment is one of the most crucial tasks in a VR application and significantly contributes to the user experience. This project focuses on the development of a physics-based character controller that seamlessly integrates into the XR Interaction Toolkit. The developed character controller expands the XR Interaction Toolkit with physics-based interaction and locomotion options.

Marching Cubes - WIP

WIP: Implementation of the marching cubes algorithm with an CPU- and GPU-based approach. The goals of this project are to explore Compute Shaders further and to create a dynamic mesh generation and manipulation system for XR applications. Consists of 2 Components the ScalarFieldGenerator and the MarchingCubesGenerator. The ScalarFieldGenerator creates a 3D scalar field based on different mathematical functions (e.g. sphere, wave ...). The MarchingCubesGenerator generates a mesh based on the scalar field using the marching cubes algorithm. The mesh is updated in real-time as the scalar field changes, allowing for dynamic and interactive manipulation of the generated shape.

Dynamic rendering of terrain information using ROAM (Real-time Optimally Adapting Meshes) and Geometry Clipmap techniques.

This project deals with the three-dimensional representation of terrain information. For this purpose, two different methods have been implemented: â€¢ Real-Time Optimal Adaptive Mesh (ROAM) method â€¢ Geometry Clipmap method The two methods pursue fundamentally different approaches to dynamically adapt the level of detail of the environment to the viewer's position. The ROAM method heavily relies on the CPU. To achieve an optimal result, the terrain mesh is adjusted every frame. However, the data structure needs to be precomputed beforehand. On the other hand, the Geometry Clipmap method largely operates on the GPU. A mesh with a constant number of triangles is built, and the height information is directly determined on the graphics card. The terrain mesh provides a higher level of detail near the viewer and accurately represents the information where necessary.

Active Assisted Living: Telephysiotherapy and barrier-free responses on paper messages using a smart home setup

"Active Assisted Living" is gaining increasing importance in Germany due to the demographic change today and in the future. "Active Assisted Living" aims to address the desires and needs of older and disadvantaged individuals, as well as the associated challenges. This paper documents the development of a system that addresses two practical issues in the field of "Active Assisted Living." This system allows for simplified email composition through analog movements and also provides a setup for automated telephysiotherapy supported by the Wii Balance Board in the comfort of one's own home. Furthermore, this work includes a guide for setting up the system.

XR · Mesh generation· Spline manipulation · Unity

MR-ider

This project explores a recreation of the Line Rider flash game mechanics from the early 2000s in XR. The user can draw splines in 3D space using a VR controller. These splines are then converted into a mesh that serves as a track for a vehicle. The vehicle's movement is governed by physics, allowing it to interact realistically with the track and environment. The goal of the project is to explore spline manipulation and mesh generation in XR.
